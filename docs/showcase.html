Problem

Factory operating decisions often sit across ERP records, WMS stock exports, spreadsheet BOM files, shipment notes, supplier updates, and informal message threads. The practical issue is not one missing dashboard; it is the time spent reconciling source rows before a release decision.

Production notices depend on template discipline, material gates, order due dates, and shop-floor assumptions. Line output and bottlenecks are difficult to discuss when takt assumptions and machine states are not connected to the same operating view.

MODULE 01

BOM & Inventory Intelligence

Finished-product demand is exploded into component demand, stock coverage, inbound quantity, supplier context, and source references. The dashboard keeps product, material, order, shipment, supplier, and WMS rows visible.

MODULE 02

Production Notice Generator

The notice preview is built from model selection, order quantity, BOM rows, material gate status, source references, and template version. The public demo exports HTML/JSON previews and keeps material watch items visible.

MODULE 03

Production Takt & Bottleneck Simulation

The simulation represents machine nodes, conveyors, robot links, buffers, running and blocked states, 24h output, utilization, waiting time, blocking time, scrap output, and bottleneck detection. Scheduling remains a stub boundary rather than a full optimizer.

MODULE 04

AI Agent Q&A

Questions are routed through registered factory tools. The Q&A panel shows user question, selected intent, workflow, tool calls, tool result, source/demo record references, and final answer. The agent is a caller and narrator, not the source of business truth.
